=== JOB MATCH EVALUATION ===

Evaluate if the job description matches the candidate profile.

**OUTPUT FORMAT (MUST FOLLOW):**

If suitable:
Yes

If NOT suitable:
No
Reason: [Brief reason]

---

**IMPORTANT: MANDATORY vs OPTIONAL SKILLS**

Only reject jobs for missing MANDATORY skills. Ignore optional skills completely.

MANDATORY skills are stated as:
- "Required", "Must have", "Should have"
- Listed under "Requirements" or "Qualifications"
- No qualifier words

OPTIONAL skills use phrases like:
- "Plus", "Bonus", "Nice to have", "Preferred", "Good to have"
- "Familiarity with", "Exposure to", "Knowledge of... advantageous"

Example:
"Required: Java, Spring Boot. Docker knowledge is a plus"
→ Evaluate ONLY Java and Spring Boot. IGNORE Docker completely.

---

**REJECT (Answer "No") if job requires:**
- Frontend, Full-stack, or UI/UX work
- Primary language other than Java (Python, Node.js, Go, etc.)
- Docker/Kubernetes as mandatory requirement
- Company size < 500 employees
- Location outside India (unless remote with IST hours)
- US/EU time zone or night shifts
- 5+ years experience required
- Senior/Lead position requiring management/mentoring
- International relocation
- Part-time or Contract positions

**ACCEPT (Answer "Yes") if job involves:**
- Backend development with Java/Spring Boot
- REST APIs, Microservices
- MySQL, MongoDB, Kafka, AWS
- 0-4 years experience required
- Location: India or Remote (IST hours)
- Company size: 500+ employees
- Roles: Software Engineer, Backend Developer, Java Developer, SDE
- Job Type: Full-time (ACCEPT full-time positions)
- Work Type: On-site, Hybrid, or Remote (ALL are acceptable)

---

**EVALUATION STEPS:**
1. Check job type (ACCEPT full-time, REJECT part-time/contract)
2. Check work type (ACCEPT on-site/hybrid/remote - ALL are suitable)
3. Check deal-breakers: company size, location, shift, role type
4. Check experience (reject if > 4 years)
5. Check primary tech (must be Java backend)
6. Check MANDATORY skills only (need 80%+ match)
7. IGNORE optional/nice-to-have skills completely

**REASON FORMAT (brief):**
- "Requires frontend development"
- "Company size < 500 employees"
- "Requires 5+ years experience"
- "Primary language is Python"
- "US time zone required"
- "Full-stack role"
- "Requires Docker/Kubernetes"

---

**OPTIONAL SKILL INDICATORS (IGNORE THESE):**
These phrases mean the skill is NOT mandatory - DO NOT reject for missing these:
- "is a plus"
- "is a bonus"
- "nice to have"
- "preferred"
- "good to have"
- "familiarity with"
- "exposure to"
- "knowledge of... would be advantageous"

Example: "Experience in Scala is a plus" → IGNORE Scala requirement completely

---

**EXAMPLES:**

Job: "Backend Java Developer, Full-time, On-site, 2-3 years, Spring Boot, MySQL, Bangalore, 1000+ employees"
Response:
Yes

Job: "Backend Java Developer, Full-time, Hybrid, 3 years, Spring Boot, Kafka, Delhi, 700 employees"
Response:
Yes

Job: "Backend Java Developer, Full-time, Remote, 2 years, Spring Boot, AWS, Mumbai, 900 employees"
Response:
Yes

Job: "Full Stack Developer, Full-time, On-site, React + Java, 2 years, Remote, 200 employees"
Response:
No
Reason: Full-stack role, company size < 500 employees

Job: "Senior Java Developer, Full-time, On-site, 5+ years, requires Docker/K8s, Pune, 800 employees"
Response:
No
Reason: Requires 5+ years experience, requires Docker/Kubernetes

Job: "Backend Engineer, Full-time, On-site, Python/Django, 2 years, Bangalore, 600 employees"
Response:
No
Reason: Primary language is Python

Job: "Java Backend Developer, Part-time, On-site, 3 years, Spring Boot, MySQL, Bangalore, 1000+ employees"
Response:
No
Reason: Part-time position

Job: "Java Backend Developer, Full-time, On-site, 3 years, Spring Boot, MySQL, Docker knowledge is a plus, Bangalore, 1000+ employees"
Response:
Yes

Job: "Java Developer, Full-time, Hybrid, 2 years, Spring Boot, AWS, experience in Kubernetes is a bonus, Mumbai, 800 employees"
Response:
Yes

---

**CRITICAL RULES:**
1. First line MUST be "Yes" or "No"
2. If "No", second line MUST be "Reason: [text]"
3. ACCEPT all full-time positions (on-site, hybrid, and remote are ALL acceptable)
4. REJECT part-time or contract positions
5. Be strict on deal-breakers (company size, location, experience, role type)
6. COMPLETELY IGNORE optional skills ("plus", "bonus", "nice to have", "preferred")
7. Only evaluate MANDATORY requirements - reject only if missing critical skills
8. Bangalore and Bengaluru are the same city
9. Do not validate job description locations against job posting location
