=== JOB MATCH EVALUATION ===

Evaluate if the job description matches the candidate profile.

**OUTPUT FORMAT (MUST FOLLOW):**

If suitable:
Yes

If NOT suitable:
No
Reason: [Brief reason]

---

**REJECT (Answer "No") if job requires:**
- Frontend, Full-stack, or UI/UX work
- Primary language other than Java (Python, Node.js, Go, etc.)
- Docker/Kubernetes as mandatory requirement
- Company size < 500 employees
- Location outside India (unless remote with IST hours)
- US/EU time zone or night shifts
- 5+ years experience required
- Senior/Lead position requiring management/mentoring
- International relocation

**ACCEPT (Answer "Yes") if job involves:**
- Backend development with Java/Spring Boot
- REST APIs, Microservices
- MySQL, MongoDB, Kafka, AWS
- 0-4 years experience required
- Location: India or Remote (IST hours)
- Company size: 500+ employees
- Roles: Software Engineer, Backend Developer, Java Developer, SDE

---

**EVALUATION STEPS:**
1. Check deal-breakers: company size, location, shift, role type
2. Check experience (reject if > 4 years)
3. Check primary tech (must be Java backend)
4. Check skill match (need 80%+)
5. Ignore nice-to-have skills

**REASON FORMAT (brief):**
- "Requires frontend development"
- "Company size < 500 employees"
- "Requires 5+ years experience"
- "Primary language is Python"
- "US time zone required"
- "Full-stack role"
- "Requires Docker/Kubernetes"

---

**EXAMPLES:**

Job: "Backend Java Developer, 2-3 years, Spring Boot, MySQL, Bangalore, 1000+ employees"
Response:
Yes

Job: "Full Stack Developer, React + Java, 2 years, Remote, 200 employees"
Response:
No
Reason: Full-stack role, company size < 500 employees

Job: "Senior Java Developer, 5+ years, requires Docker/K8s, Pune, 800 employees"
Response:
No
Reason: Requires 5+ years experience, requires Docker/Kubernetes

Job: "Backend Engineer, Python/Django, 2 years, Bangalore, 600 employees"
Response:
No
Reason: Primary language is Python

---

**CRITICAL RULES:**
1. First line MUST be "Yes" or "No"
2. If "No", second line MUST be "Reason: [text]"
3. Be strict on deal-breakers
4. Be lenient on nice-to-have skills
5. Bangalore and Bengaluru are the same city
6. Do not validate job description locations against job posting location
