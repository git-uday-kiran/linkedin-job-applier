=== JOB MATCH EVALUATION INSTRUCTIONS ===

You will be provided with a job description. Evaluate if this job is suitable for the candidate based on the profile context provided earlier.

**INPUT:**
Job Description: {{JOB_DESCRIPTION}}

**OUTPUT FORMAT:**
You MUST respond in one of these two formats ONLY:

**Format 1 - If job is suitable:**
Yes

**Format 2 - If job is NOT suitable:**
No
Reason: [Brief reason why not suitable]

---

**EVALUATION CRITERIA:**

**MUST REJECT (Answer "No") if job requires:**
- Frontend development, Full-stack development, or UI/UX work
- Technologies NOT in profile: Docker, Kubernetes, Redis, PostgreSQL, TDD
- Work location outside India (unless remote with IST hours)
- US/EU time zone shifts or night shifts
- Company size less than 500 employees
- Experience level significantly higher than 2+ years (e.g., 5+ years required)
- Senior/Lead positions requiring mentoring or management experience
- Programming languages other than Java as primary language
- International relocation required

**SHOULD ACCEPT (Answer "Yes") if job involves:**
- Backend development with Java/Spring Boot
- REST APIs, Microservices architecture
- MySQL, MongoDB databases
- Technologies in profile: Java, Spring Boot, Spring Security, Kafka, AWS
- Experience requirement: 0-4 years
- Location: India (any city) or Remote
- Company size: 500+ employees
- IST working hours
- Product-based or startup companies
- Roles: Software Engineer, Backend Developer, Java Developer, SDE

**EVALUATION PROCESS:**
1. Check mandatory deal-breakers first (company size, location, shift, role type)
2. Check required experience level (reject if too senior)
3. Check primary technology stack (must be Java-focused backend)
4. Check required skills vs profile skills (80%+ match needed)
5. Consider nice-to-have skills (don't reject for these alone)

**REASON FORMAT:**
Keep reason brief and specific. Examples:
- "Requires frontend development"
- "Company size less than 500 employees"
- "Requires 5+ years experience"
- "Requires Docker/Kubernetes expertise"
- "US time zone required"
- "Full-stack role, not backend-only"
- "Primary language is Python, not Java"
- "Senior position requiring team lead experience"

---

**RESPONSE EXAMPLES:**

Example 1:
Job: "Backend Java Developer, 2-3 years exp, Spring Boot, MySQL, Bangalore, 1000+ employees"
Response:
Yes

Example 2:
Job: "Full Stack Developer, React + Java, 2 years exp, Remote, 200 employees"
Response:
No
Reason: Full-stack role (requires frontend), company size less than 500 employees

Example 3:
Job: "Senior Java Developer, 5+ years, Spring Boot, AWS, requires Docker/K8s, Pune, 800 employees"
Response:
No
Reason: Requires 5+ years experience, requires Docker/Kubernetes expertise

Example 4:
Job: "Backend Engineer, Python/Django, 2 years, Bangalore, 600 employees"
Response:
No
Reason: Primary language is Python, not Java

Example 5:
Job: "Java Backend Developer, 1-3 years, microservices, night shift US hours, 1500 employees"
Response:
No
Reason: Requires US time zone night shift

---

**CRITICAL RULES:**
1. Always answer "Yes" or "No" on first line
2. If "No", provide "Reason:" on second line
3. Be strict on deal-breakers (company size, location, shift, role type)
4. Be lenient on nice-to-have skills if core match is good
5. Prioritize candidate's mandatory requirements and preferences
